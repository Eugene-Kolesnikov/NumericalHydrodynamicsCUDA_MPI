◆ DLV_init()

bool DLV_init ( size_t  N_X,
size_t  N_Y,
enum OUTPUT_OPTION  outOption 
)

The server node calls the DLV_init function once before the computations start in order to initialize the environment: create the window where OpenGL will render the field, allocate necessary memory, initialize environmental variables, set the output method (PPM, PNG, MPEG).

Parameters
N_X– discretization of the grid along the X direction.
N_Y– discretization of the grid along the Y direction.
outOption– parameter specifies what output is expected from the DLV: 'PPM' creates a .ppm image file, 'PNG' creates a .png image file, 'MPEG' creates a .mpeg video file. In the last case, the function initializes the file for further writing frames into it.
Returns
a bool parameter which indicates if the function performed successfully (true) or not (false).

Create the window with an OpenGL context of the 'MainWindow' class

Set up the background color of the window (Dark blue)

Allocate memory for the FieldObject

Set up the output method

Initialize the FiledObject

Return FAIL if any runtime exception caught

bool DLV_visualize ( void *  field,
size_t  N_X,

The server node calls the DLV_visualize function each time it is necessary to plot hydrodynamics fields: density, velocity,... (depends on the computational model). The decision of what property of the field must be rendered is specified in the DLV library.

Parameters
field– pointer to the field.
N_X– discretization of the grid along the X direction.
N_Y– discretization of the grid along the Y direction.
Returns
a bool parameter which indicates if the function performed successfully (true) or not (false).

If the window shouldn't be closed, clear the screen and render the field

Return FAIL if the window should be closed but the rendering is required
